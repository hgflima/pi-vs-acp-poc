**New function: startOAuth(provider)**
1. Set provider state to `{ status: "connecting", authMethod: null, error: null }`
2. Call `startOAuthApi(provider)`
3. If response status is "error", set state to `{ status: "error", authMethod: null, error: res.message || "OAuth flow failed to start" }` and return
4. If response status is "started" and `authUrl` exists:
   a. Open popup: `window.open(res.authUrl, '_blank', 'width=600,height=700,popup=yes')`
   b. If popup is null (blocked), set state to `{ status: "error", authMethod: null, error: "Pop-up blocked. Please allow pop-ups for this site and try again." }` and return
   c. Start polling with `setInterval` every 2000ms:
      - Call `fetchOAuthStatus(provider)`
      - If status is "ok": clear interval, remove from pollingRef, call `refreshStatus(provider)` to sync auth state from backend
      - If status is "error": clear interval, remove from pollingRef, set state to `{ status: "error", authMethod: null, error: data.message || "OAuth authorization failed" }`
      - If status is "pending" or "none": continue polling (no action)
   d. Store interval ID in `pollingRef.current.set(provider, intervalId)`

**CRITICAL — Pitfall 4 from RESEARCH.md:** Backend auto-deletes session on first read after "ok"/"error". Stop polling IMMEDIATELY on terminal state. Do NOT retry after receiving "ok" or "error".

**CRITICAL — Pitfall 2 from RESEARCH.md:** The setInterval callback captures stale closures. Use a ref-based cancelled flag OR use a recursive setTimeout chain. The simplest safe approach: inside the interval callback, check if the interval is still in pollingRef before acting on the result.

**New function: cancelOAuth(provider)**
1. Get interval from `pollingRef.current.get(provider)`
2. If exists: `clearInterval(interval)`, `pollingRef.current.delete(provider)`
3. Set state to `{ status: "disconnected", authMethod: null, error: null }` (D-06: reset to initial state, orphaned backend session expires naturally)

**Enhanced disconnect(provider):**
Replace the current local-only disconnect with:
```typescript
const disconnect = useCallback(
  async (provider: Provider) => {
    try {
      await disconnectProvider(provider)
    } catch {
      // Best-effort — clear local state regardless
    }
    // Also cancel any active polling for this provider
    const interval = pollingRef.current.get(provider)
    if (interval) {
      clearInterval(interval)
      pollingRef.current.delete(provider)
    }
    setProviderState(provider, { status: "disconnected", authMethod: null, error: null })
  },
  [setProviderState]
)
```
