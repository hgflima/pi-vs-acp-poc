## Tool Card Visual Specification

### Common Structure (all variants)

```
+-- border-radius: rounded-lg (8px) ---------------------------------+
| HEADER: icon + label + tool-specific info + ToolStatusIcon    py-1.5 px-3 |
+---------------------------------------------------------------------+
| CONTENT: variant-specific content area                     py-2 px-3 |
|   (only visible when result/output exists or status != running)      |
|   max-height: 256px (max-h-64), overflow-y: auto                    |
+---------------------------------------------------------------------+
```

- **Vertical margin:** `my-3` (12px above and below each tool card)
- **Border radius:** `rounded-lg` (0.5rem / 8px)
- **Overflow:** `overflow-hidden` on outer container
- **Content max-height:** `max-h-64` (256px) with `overflow-auto` for scrollable output

### Bash Card Layout

```
+-- bg-zinc-900, border border-zinc-700, rounded-lg ------------------+
| [Terminal icon]  Terminal                      [StatusIcon]  bg-zinc-800 |
+---------------------------------------------------------------------+
| $ {command}                                          font-mono text-sm |
+-- border-t border-zinc-700 (only if result exists) -----------------+
| {output}                        font-mono text-xs text-zinc-400      |
|                                 max-h-64 overflow-auto               |
+---------------------------------------------------------------------+
```

- Header background: `bg-zinc-800`
- Body background: `bg-zinc-900` (matches `#0d1117` github-dark)
- Command prefix: `$ ` in `text-zinc-500`
- Command text: `text-zinc-200 font-mono text-sm`
- Output text: `text-zinc-400 font-mono text-xs whitespace-pre-wrap`
- Error output: `text-red-400` instead of `text-zinc-400`

### File Card Layout

```
+-- bg-green-50, border border-green-200, rounded-lg -----------------+
| [FileText icon]  {file_path}                  [StatusIcon]  text-xs  |
+---------------------------------------------------------------------+
| {file content with syntax highlighting}       font-mono text-sm      |
|                                 max-h-64 overflow-auto               |
+---------------------------------------------------------------------+
```

- Icon: FileText (lucide) in `text-green-600`
- File path: `font-mono text-xs truncate` — truncated if long
- Content: reuse CodeBlock component or inline highlight.js for syntax highlighting
- Write variant: show "Wrote {n} bytes" summary if no content preview available

### Search Card Layout

```
+-- bg-blue-50, border border-blue-200, rounded-lg -------------------+
| [Search icon]  {pattern}                      [StatusIcon]  text-xs  |
+---------------------------------------------------------------------+
| {result line 1}                               font-mono text-xs      |
| {result line 2}                                                       |
| ...                                           max-h-64 overflow-auto |
+---------------------------------------------------------------------+
```

- Icon: Search (lucide) in `text-blue-600`
- Pattern/query: `font-mono text-xs`
- Results: each line in `font-mono text-xs text-foreground`, separated by newlines

### Agent Card Layout

```
+-- bg-orange-50, border border-orange-200, rounded-lg ---------------+
| [Bot icon]  {agent_name}                      [StatusIcon]  text-xs  |
+---------------------------------------------------------------------+
| {result or progress text}                     text-sm text-foreground |
|                                 max-h-64 overflow-auto               |
+---------------------------------------------------------------------+
```

- Icon: Bot (lucide) in `text-orange-600`
- Agent name: `text-xs font-medium`
- Running state: shows agent name + spinning status icon only (no content area until result arrives)

### Toolsearch Card Layout

```
+-- bg-muted, border border-border, rounded-lg -----------------------+
| [Wrench icon]  Tool Search                    [StatusIcon]  text-xs  |
+---------------------------------------------------------------------+
| {tool1 name}                                  text-sm                |
| {tool2 name}                                                         |
| ...                                           max-h-64 overflow-auto |
+---------------------------------------------------------------------+
```

- Icon: Wrench (lucide) in `text-muted-foreground`
- Tool names: listed as `text-sm` items, one per line

### Generic Card Layout

```
+-- bg-muted, border border-border, rounded-lg -----------------------+
| [Cog icon]  {tool_name}                       [StatusIcon]  text-xs  |
+---------------------------------------------------------------------+
| Params: {JSON.stringify(args, null, 2)}       font-mono text-xs      |
| ---                                                                   |
| Result: {result text}                         font-mono text-xs      |
|                                               max-h-64 overflow-auto |
+---------------------------------------------------------------------+
```

- Icon: Cog (lucide) in `text-muted-foreground`
- Tool name: `text-xs font-medium`
- Params section: `font-mono text-xs text-muted-foreground whitespace-pre-wrap`
- Result section: `font-mono text-xs text-foreground whitespace-pre-wrap`

## Interaction Contract

### Tool Card Lifecycle (Real-Time Updates)

| Event | UI Change | Timing |
|-------|-----------|--------|
| `tool_start` SSE | New tool card appears inline in segments with "running" status, spinning icon | Immediate — card renders at TOOL_START, do not wait for result |
| `tool_update` SSE | Content area updates with streaming partial result | Append/replace result text as updates arrive |
| `tool_end` SSE | Status changes to done/error, final result displayed, icon stops spinning | Immediate status transition |
| Text after tool | New TextSegment created, renders as markdown below the tool card | Automatic via existing APPEND_TEXT_DELTA handler |

### Auto-Scroll Behavior

Tool cards participate in the existing auto-scroll mechanism (Phase 2). When a new tool card appears or updates, the chat area scrolls down if the user has not manually scrolled up. No changes to the 50px threshold or userScrolledUp ref.

### No Expand/Collapse in Phase 3

Expand/collapse is deferred to v2 (TOOLX-01). All tool cards render their full content area (up to max-h-64 scroll limit). The chevron toggle documented in SCR-02 wireframes is out of scope for this phase.

### Multiple Concurrent Tools

Multiple tool segments can have `status: "running"` simultaneously. Each is identified by `toolId`. The reducer looks up segments by ID, not by position.
