### JSON Response Envelope Shape (active contract)

All Phase 7 endpoints use the established envelope from Phase 6:

```json
{ "status": "ok" | "error" | "pending" | "started" | "none", "message": "...", "provider": "openai", "authUrl": "..." (when applicable) }
```

- `status: "started"` — returned by `POST /start` with `authUrl` populated
- `status: "pending"` — returned by `GET /status` while OAuth session in flight
- `status: "done"` — returned by `GET /status` after `storeOAuthTokens` completes
- `status: "error"` — returned on port conflict (409) or OAuth rejection (500)
- `status: "none"` — returned by `GET /status` when no session exists

**Never echo `access` or `refresh` token values in any response body** (CLAUDE.md security constraint). Debug force-expire response MAY include `before`/`after` expiry epoch numbers (D-06 discretion), but NEVER the token strings.

### Log Messages (operator-facing)

`onProgress` messages from `loginOpenAICodex` are logged to console with prefix `[oauth:openai]`. Format: `[oauth:openai] {message}` (RESEARCH.md Pattern 1 example).
