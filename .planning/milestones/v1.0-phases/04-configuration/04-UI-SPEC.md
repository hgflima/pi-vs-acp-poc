# Phase 4 — UI Design Contract

> Visual and interaction contract for the Configuration phase. Agent switching, model switching, and harness loading UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI v4) |
| Preset | base-nova (neutral base, CSS variables, no RSC) |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react ^1.7.0 |
| Font | Geist Variable (sans), JetBrains Mono (mono) |

**Source:** `components.json` preset + `globals.css` `@import "@fontsource-variable/geist"`. Design tokens in `.harn/design/design-tokens.json`.

## Spacing Scale

Inherited from master design tokens. All values are multiples of 4.

| Token | Value | Usage in Phase 4 |
|-------|-------|-------------------|
| spacing.1 | 4px | Icon-to-text gap inside popover items, badge internal padding |
| spacing.2 | 8px | Popover item vertical padding, inline-auth field gap |
| spacing.3 | 12px | Popover section divider margin, settings section inner padding |
| spacing.4 | 16px | Popover content padding, settings card padding, form field gap |
| spacing.6 | 24px | Settings page section gap, page padding |
| spacing.8 | 32px | Settings page top/bottom margin |
| spacing.12 | 48px | Settings page max-width centering |

**Exceptions:**
- `spacing.3 = 12px` -- inherited from master design-tokens.json. Grid-aligned (3x4px). Used for component-level micro-gaps where 8px is too tight and 16px is too loose: separator margin in popover, agent/model item padding-x, harness badge gap, settings subtitle margin.

## Color

Inherited from shadcn base-nova neutral theme + master design tokens.

| Role | Value | Usage in Phase 4 |
|------|-------|-------------------|
| Dominant (60%) | `--background` (white) | Settings page background, popover background |
| Secondary (30%) | `--secondary` / `--muted` | Popover selected item highlight, settings section card bg, model badge bg in header |
| Accent (10%) | `--primary` (near-black in base-nova neutral) | Active agent indicator dot, "Save API Key" button in inline auth, "Load Harness" button |
| Destructive | `--destructive` (oklch red) | Error states only: harness loading errors, auth failure |

**Dual token system note:** The project has two color sources: (1) `design-tokens.json` which defines a warm terracotta accent (`color.accent.500: #E07A5F`) intended for future branding, and (2) the shadcn base-nova theme which defines `--primary` as near-black. Phase 4 uses only the shadcn `--primary` token for interactive elements (buttons, selection states). The terracotta accent from `design-tokens.json` is not used in Phase 4 components. Semantic status colors (`color.success.*`, `color.warning.*`, `color.error.*`) are sourced from `design-tokens.json` as they have no shadcn equivalent.

## Component Specifications

### AgentModelPopover

**Trigger:** Clickable region in header showing agent icon + agent name + chevron-down icon. The entire trigger area is a single click target (D-01, Cursor-style).

```
Header trigger (rest):
+------------------------------+
| [Bot] Claude Code  [ChevronDown] |  [claude-sonnet-4 badge]  [Settings icon]
+------------------------------+

Popover content (open):
+----------------------------------+
|  AGENT                           |  <- section label, 12px, uppercase, muted
|                                  |
|  [Bot]  Claude Code        [*]   |  <- selected: green dot
|  [Cpu]  Codex                    |  <- unselected: no dot
|                                  |
|  --------------------------------|  <- separator
|                                  |
|  MODEL                           |  <- section label
|                                  |
|  claude-sonnet-4-20250514  [*]   |  <- selected model
|  claude-opus-4-20250514          |
|  claude-haiku-4-20250514         |
|  ...                             |  <- scrollable if > 6 models
|                                  |
+----------------------------------+
```

**Layout:**
- Popover width: 280px
- Popover max-height: 400px (overflow-y auto on model section)
- Content padding: spacing.4 (16px)
- Section label: 12px, weight 500, uppercase, `--muted-foreground`, margin-bottom spacing.2
- Agent item height: 36px, padding-x spacing.3, border-radius `--radius-md`
- Model item height: 32px, padding-x spacing.3, border-radius `--radius-md`
- Separator: 1px `--border`, margin-y spacing.3
- Agent icon: 16px, `--muted-foreground`
- Selected indicator: 8px green dot (`color.success.500`), right-aligned

**States:**
- Agent item default: transparent bg, `--foreground` text
- Agent item hover: `--accent` bg
- Agent item selected: `--accent` bg, green dot visible
- Model item default: transparent bg, `--foreground` text, 14px
- Model item hover: `--accent` bg
- Model item selected: `--accent` bg, green dot visible
- Models loading: 3 skeleton lines (pulse animation), 32px height each
- Provider not authenticated: model section replaced by InlineAuth component

**Interactions:**
- Click agent item: switch agent, clear chat (D-05/D-06), close popover, refetch models
- Click model item: switch model, clear chat (D-05/D-06), close popover
- Escape: close popover
- Click outside: close popover
- Popover closes on every selection (Pitfall 5 defense)

## Interaction Contracts

### Agent Switch Flow (D-01, D-05, D-06)

1. User clicks agent name in header (PopoverTrigger)
2. Popover opens with agent list + model list
3. User clicks a different agent
4. Immediately: popover closes, chat messages clear, header updates to new agent icon + name
5. Model badge shows loading state (skeleton or "...")
6. Backend GET `/api/models?provider={new_provider}` is called
7. If provider authenticated: model list loads, default model selected, badge updates
8. If provider NOT authenticated (401): popover reopens with InlineAuth in place of model section
9. After auth success: model list loads normally

### Model Switch Flow (D-05, D-06)

1. User clicks agent name in header (PopoverTrigger)
2. Popover opens with agent list + model list
3. User clicks a different model
4. Immediately: popover closes, chat messages clear, model badge updates to new model name

### Inline Auth Flow (D-10, D-11)

1. User switches to agent whose provider lacks credentials
2. Popover shows InlineAuth where model section would be
3. User enters API key, clicks "Save API Key"
4. Button shows "Saving..." with spinner
5. Backend validates key (reuse existing POST `/api/auth` endpoint)
6. On success: InlineAuth replaced by model list, default model auto-selected
7. On error: error text appears below input, user can retry

### Harness Loading Flow (D-07, D-08, D-12, D-13)

1. User clicks Settings icon in header or harness badge
2. Navigates to `/settings` page
3. User drops folder or enters directory path
4. Frontend sends directory path to backend POST `/api/harness/load`
5. Backend discovers files at known locations (CLAUDE.md, AGENTS.md, .claude/skills/, .claude/hooks/)
6. Response shows per-file status in the discovered files list
7. User clicks "Load Harness"
8. Backend stores harness content in memory, applies to agent system prompt
9. On success: toast shown, navigate back to `/chat`, harness dot visible in header
10. On error (per file): inline error on that file row, other files still loadable (D-12)
11. Harness persists in memory until next load or server restart (D-13)

### Navigation Flows

```
/chat  --[click settings icon]-->  /settings
/settings  --[click "Back to Chat"]-->  /chat
/settings  --[load success]-->  /chat (automatic redirect)
```

## Accessibility

### Popover (AgentModelPopover)
- `role="dialog"` on PopoverContent (Radix default)
- Agent items: `role="radio"`, `aria-checked` for selected
- Model items: `role="radio"`, `aria-checked` for selected
- Focus moves to first agent item on open
- Arrow keys navigate items within each section
- Enter/Space selects item
- Escape closes popover, returns focus to trigger
- Tab cycles through sections and close button

### Inline Auth
- Input: `aria-label="API key"`, `type="password"`
- Error: `aria-live="polite"` on error region
- Button: `aria-busy="true"` during saving

### Settings Page
- Page: `aria-labelledby` pointing to heading
- Drop zone: `role="button"`, `aria-label="Drop project folder or click to browse"`
- File status list: `role="list"`, each row `role="listitem"`
- Status icons: `aria-label` describing status ("Loaded", "Not found", "Error")
- Load Harness button: `aria-busy="true"` during loading

### Focus Management
- Popover open: focus to first agent item
- Popover close: focus returns to trigger button
- Navigate to /settings: focus to "Back to Chat" link
- Load success + redirect: focus to chat input
