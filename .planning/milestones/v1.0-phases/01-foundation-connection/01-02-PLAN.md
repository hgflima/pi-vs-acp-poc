    Create the spike script following Research Pattern 5 exactly. The script must:

    1. **Import and define a test tool** using `Type` from pi-ai for parameter schema:
       ```typescript
       import { Agent } from "@mariozechner/pi-agent-core"
       import { getModel, streamSimple, Type } from "@mariozechner/pi-ai"
       import type { AgentTool } from "@mariozechner/pi-agent-core"
       ```

       Define an `echoTool` with:
       - name: "echo"
       - label: "Echo"
       - description: "Echoes back the given message"
       - parameters: `Type.Object({ message: Type.String({ description: "Message to echo" }) })`
       - execute: async function returning `{ content: [{ type: "text", text: "Echo: ${params.message}" }], details: {} }`

    2. **Create Agent** with:
       - `initialState.systemPrompt`: "You have an echo tool. Use it to echo 'hello world'."
       - `initialState.model`: `getModel("anthropic", "claude-haiku-3-5")`
       - `initialState.tools`: `[echoTool]`
       - `initialState.thinkingLevel`: "off"
       - `streamFn`: `streamSimple`

    3. **Subscribe to events** and collect event types:
       - `agent_start` -> log "[spike] Agent started"
       - `message_update` -> if `event.assistantMessageEvent?.type === "text_delta"`, write delta to stdout
       - `tool_execution_start` -> log tool name and args
       - `tool_execution_end` -> log result status (OK or ERROR)
       - `agent_end` -> log "[spike] Agent finished"

    4. **Run the prompt**: `await agent.prompt("Please use the echo tool to echo 'hello world'")`
       - NOTE: The third parameter of `agent.prompt()` may accept options including `{ apiKey }`. If `agent.prompt` does NOT accept apiKey, pass it via the Agent constructor options or `streamFn` wrapper. The Research shows `streamSimple` accepts `{ apiKey }` as third param -- so if Agent's `streamFn` is `streamSimple`, the apiKey must be passed through the Agent's streamFn configuration.
       - **Strategy for API key:** Read `process.env.ANTHROPIC_API_KEY`. If the Agent constructor or streamFn doesn't accept apiKey directly, create a wrapper around streamSimple that injects the key:
         ```typescript
         const apiKey = process.env.ANTHROPIC_API_KEY
         if (!apiKey) {
           console.error("[spike] FAIL: Set ANTHROPIC_API_KEY env var")
           process.exit(1)
         }

         // Wrapper that injects apiKey into streamSimple
         const streamFnWithKey: typeof streamSimple = (model, context, options) => {
           return streamSimple(model, context, { ...options, apiKey })
         }
         ```
         Then pass `streamFnWithKey` as the Agent's `streamFn`.

    5. **Verify lifecycle** after prompt completes:
       - Required events: `["agent_start", "tool_execution_start", "tool_execution_end", "agent_end"]`
       - If any missing: `console.error("[spike] FAIL: Missing events: ...")` and `process.exit(1)`
       - If all present: `console.log("[spike] PASS: All required events observed")` and print event sequence

    **IMPORTANT:** This spike uses a REAL API key and makes a REAL API call. The executor should:
    - Check that `ANTHROPIC_API_KEY` is set in the environment before running
    - If not set, the script should exit with a clear error message
    - The test request is minimal (haiku model, short prompt) so cost is negligible (~$0.001)

    **If the pi-agent-core API doesn't match the Research patterns:**
    - Document the actual API in the spike's console output
    - Adapt the script to work with the real API
    - Note discrepancies in the SUMMARY for future phases to reference
